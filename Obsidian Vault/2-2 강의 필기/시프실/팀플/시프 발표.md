다음은 카메라 파이의 소프트웨어 구조 및 설계입니다. 카메라 파이는 전체적으로 이미지 촬영 및 전처리, 셀 데이터 추출, LED 상태 업데이트로 구성됩니다

- 이미지 촬영 및 전처리 - saveSudokuCells()
	- fork()로 프로세스 복제
	- 자식 프로세스에서 system()을 통해 파이썬 스크립트(sudoku_cells_extractor.py)를 호출하여 이미지 전처리 수행
	- 이미지 전처리 후 결과값(성공 시 0, 실패 시 1) 반환
	- pipe()로 자식 프로세스의 결과값을 리다이렉트하여 부모 프로세스에 전달, 부모 프로세스에서 이미지 전처리 성공 여부 확인

- 이미지 전처리의 성공 여부에 따라 LED로 피드백을 제공
- 실패 시 재시도 로직을 통해 다시 이미지 촬영 후 전처리를 시도

- 이미지 전처리 성공 시 스도쿠 데이터 추출 - getCellsNum()
- popen()을 통해 파이썬 스크립트(text_getter.py)를 호출하여 전저리된 이미지로부터 숫자를 인식 후 출력
- 인식한 숫자를 문자열로 가공하여 서버로 전송

---

- 기존 파이썬 베이스 데이터베이스 설계에서 c언어 기반의 SQLite로 변경하여 c언어 기반으로 서버 코드를 통일

- 기존 grayscale 기반의 단순한 이미지 전처리 과정에서 블러링, 적응형 이진화, 원근 변환을 추가하여 이미지 전처리를 개선
- 숫자 인식에서 추가적인 블러링 처리를 한 후 각 숫자 셀의 중간 영역 추출로 외곽선을 제외시킴으로서 숫자 인식 정확도 개선

- 기존 멀티스레드 구조의 설계에서 통합 스레드 구조로 변경
- 스도쿠 문제라는 input에 대해서 숫자 인식 및 풀이, 데이터베이스에 저장, 진행 흐름에 따른 입출력 파이과 서버 파이의 정보 교환에 의한 LCD 화면 표시 라는 하나의 큰 흐름으로 이어지므로 모든 클라이언트를 종합적으로 처리하는 통합 스레드가 더 적합하다고 생각하여 변경
