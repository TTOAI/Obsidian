- 이벤트 기반 오케스트레이션(Event-driven orchestration) 서비스
- 특정 이벤트가 발생했을 때, 자동으로 다른 서비스를 실행하게 만들어줌

---

### 구성 요소

#### Event
- 이벤트소스에서부터 오는 이벤트
#### Event Bus
- 이벤트를 받는 역할
- event rule을 생성할 때 event bus에 연결해야 함
- Default Event Bus: AWS account마다 기본적으로 생성되는 event bus
#### Event Rule
- event rule과 매칭되는 event만 event rule에 연결된 event target으로 보냄
#### Event Target
- event rule의 필터링을 통과한 event가 도착하는 대상
- 하나의 event rule에 여러 개의 event target 등록 가능

### EventBrindge Scheduler

- EventBridge 안에서 시간 기반 트리거(Scheduler)만 전문적으로 다루는 독립형 서비스
- 특정 시간 또는 주기(cron/interval)에 이벤트를 발생시키는 서비스
- 스케줄 표현식
	- Rate
	- Cron

#### Cron
- UNIX 계열 시스템에서 돌아가는 작업 스케줄러 데몬(daemon)
- 시스템마다 구현 방식이나 쓰이는 특수문자가 다를 수 있음
- cron 표현식 구조: `분 시 일 월 요일 <command>`
- AWS에서는 UTC 기준이므로, KST = UTC + 9 를 항상 고려해야 함
- 필드 수:
	- 전통 UNIX cron: 5개
	- AWS EventBridge cron: 6개 (+연도)
- 특수문자
	- `*`: 모든 값
	- `?`: 아무 것도 지정 안 함
		- 특정 필드에서만 사용: `(일)Day-of-month`, `(일)Day-of-week`
		- 올바른 예시) `cron(0 4 ? * MON *)`: 매주 월요일 4시
		- 틀린 예시) `cron(0 4 * * MON *)`: 매일 4시
	- `,`: 여러 값
	- `-`: 범위
	- `/`: 간격
	- `L`: 마지막
	- `#`: n번째 요일
	- `W`: 가까운 평일
		- ex) `15W` → 15일 기준 가장 가까운 평일 실행
	- `%`: command 내 줄바꿈. 이후의 텍스트는 표준 입력으로 전달
- 예시:
	- 매일 오전 9시: `cron(0 9 * * ? *)`
	- 매일 자정(0시): `cron(0 0 * * ? *)`
	- 매주 월요일 오전 6시: `cron(0 6 ? * MON *)`
	- 평일 오전 8시: `cron(0 8 ? * MON-FRI *)`
	- 매시간 정각: `cron(0 * * * ? *)`
	- 10분마다: `cron(0/10 * * * ? *)`
	- 매월 1일 자정: `cron(0 0 1 * ? *)`

#### 유연한 기간
- Scheduler는 지정한 시간 범위 내에서 일정을 호출
- 동시 트리거 폭주 방지를 위해 설정
	- Lambda 동시 실행 폭주
	- Step Functions 과부하
	- CloudWatch / DynamoDB / RDS 비용 급증
	- 네트워크 일시적 병목
