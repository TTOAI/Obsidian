
### RUN

- 이미지 빌드 시 실행 (`docker build`)
- 이미지에 **영구적인 변경사항**을 적용하고 환경을 설정
	- 패키지 설치, 파일 생성, 환경 변수 설정
- `RUN` 명령마다 새로운 이미지 레이어가 생성되어 이미지에 저장됨

### ENTRYPOINT

- 컨테이너 실행 시 실행 (`docker run`)
- 컨테이너의 진입점
- 컨테이너를 특정 실행 파일(실행 도구)처럼 동작하게 만들 때 사용
	- ex) 컨테이너를 CLI 도구처럼 사용
- `ENTRYPOINT`의 인자로 전달되는 명령어
	1. `docker run` 뒤에 입력된 명령어
	2. `CMD`로 정의된 명령어 -> `1.`로 덮어쓰여질 수 있음

### CMD

- 컨테이너 실행 시 실행 (`docker run`)
- 컨테이너가 시작될 때 실행할 기본 작업을 정의
- `ENTRYPOINT`에게 전달할 기본 인자로 쓰일 수 있음
- 사용자가 `docker run` 명령 뒤에 다른 명령어를 입력하면 `CMD`에 정의된 기본 명령어는 무시되고 덮어쓰여짐
