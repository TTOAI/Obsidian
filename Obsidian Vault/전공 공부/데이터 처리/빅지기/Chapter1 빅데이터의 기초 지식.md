- 빅데이터 세계에서 사용되는 기술과 그 역할을 차례대로 살펴보는 구성

## 1-1 [배경] 빅데이터의 정착

- 데이터 처리에 분산 시스템 도입
- 빅데이터라는 용어가 자주 쓰이게 됨

- 빅데이터의 취급이 어려운 이유
	- 데이터의 분석 방법을 모름
	- 데이터 처리에 수고와 시간이 걸림

- 대표적인 빅데이터 기술: Hadoop, NoSQL

- 웹 서버 등에서의 데이터 -> RDB, NoSQL, ... -> Hadoop -> 대규모 처리 시작

- Hadoop: 다수의 컴퓨터에서 대량의 데이터 처리 및 집계
	- Hive의 등장으로 Hadoop에서 SQL을 쓸 수 있게 되면서 사용자 확대
> Hadoop에서 데이터를 처리하고 집계한다는 게 정확히 어떤 의미?
---
- NoSQL: 고속 읽기/쓰기 가능 및 분산 처리에 뛰어남
	- 종류:
		- 키 밸류 스토어
		- 도큐멘트 스토어
		- 와이드 칼럼 스토어
---
- 데이터 웨어하우스 + Hadoop
	- 가속도적으로 늘어나는 데이터의 처리 -> Hadoop
	- 비교적 작은 데이터 또는 중요한 데이터 -> 데이터 웨어하우스
> 데이터 웨어하우스가 정확히 뭐임?

> RDB와 DW 사이에 Hadoop을 두면 왜 DW의 크기가 작아짐? 
---
- 클라우드 서비스의 등장으로 작은 프로젝트 단위에서도 DW 구축이 보편화
---
- 데이터 디스커버리: DW에 저장된 데이터를 대화형으로 시각화하여 가치 있는 정보를 찾으려고 하는 프로세스 = 셀프 서비스용 BI 도구

## 1-2 빅데이터 시대의 데이터 분석 기반

- 데이터 전송의 방법
	- 벌크 형: 정기적으로 데이터를 정리 및 추출
	- 스트리밍 형: 생성되는 데이터를 끊임없이 계속해서 보내는 방법

- 모바일 애플리케이션의 증가 -> 스트리밍 형이 주류
- 스트리밍 형의 실시간 처리 -> 스트림 처리
- 스트림 처리의 결과를 시계열 데이터베이스에 저장

- 장기적인 데이터 분석 ->정리된 데이터의 효율적 가공 필요 -> 배치 처리
---
- 스트림 처리 및 배치 처리로 수집된 데이터 -> 분산 스토리지에 저장
- 분산 스토리지: 여러 컴퓨터와 디스크로부터 구성된 스토리지 시스템
- 데이터를 저장하는 대표적인 방법: 객체 스토리지 (ex. S3)
- NoSQL 데이터베이스를 분산 스토리지로 사용 가능
---
- 분산 데이터 처리:
	- 분산 스토리지에 저장된 데이터를 처리
	- 데이터 가공 및 외부 데이터베이스에 저장

- 데이터 집계 방법:
	- 쿼리 엔진 (ex. Hive, 대화형 쿼리 엔진)
	- 외부 DW 이용 -> 데이터를 DW에 적합한 형식으로 변환 필요 -> ETL 프로세스 필요
		- ETL 프로세스: 데이터 추출, 가공, 로드
---
- 워크플로 관리:
	- 전체 데이터 파이프라인 동작 관리
	- 데이터 처리를 자동화해서 장기적으로 운용해 나가기 위해 필수적임
---
- 업무용 RDB, 로그 등을 저장 -> 데이터 소스
- 데이터 소스에서 로우 데이터 추출, 가공, DW에 저장 -> ETL 프로세스
---
- 데이터 마트(DM): 데이터 분석 등을 위해 DW에서 필요한 데이터만을 추출
---
- 데이터 레이크(DL):
	- 모든 데이터를 원래의 형태로 축적
	- 임의의 테이터를 저장할 수 있는 분산 스토리지
	- 대부분 CSV, JSON 등의 형식
---
- 애드 혹 분석:
	- 수작업을 통한 일회성 데이터 분석
	- 데이터 마트를 만들지 않은 채 DL과 DW에 직접 연결
- 대시보드 도구:
	- 그래프와 보고서를 만들 때 사용
	- 데이터 마트가 없어도 동작함
	- 설정한 스케줄에 따라 쿼리 실행
---
- 데이터 수집 목적:
	- 데이터 검색
		- 실시간 데이터 처리나 검색 엔진을 사용하여 키워드를 찾는 기능 필요
	- 데이터 가공
		- 데이터 처리 결과를 업무 시스템에 이용
		- 데이터 파이프라인 설계
		- 자동화, 워크플로 관리 필수
	- 데이터 시각화
		- 데이터 시각화를 통해 상황 예측 및 의사 결정
		- 고속화하려면 데이터 마트 필요
---
- 기간계 시스템:
	- 비즈니스 근간에 관련된 중요한 시스템
	- 완벽하게 테스트 반복 및 신중하게 운용
	- 실행 과정을 로그 파일이나 데이터베이스 등에 기록
- 정보계 시스템:
	- 사내 커뮤니케이션과 의사 결정 등을 위해 이용하는 시스템
	- 데이터를 효율적으로 분석하려면 정보계 시스템으로 분리

- 데이터: 기간계 시스템과 정보계 시스템을 연결하는 것
