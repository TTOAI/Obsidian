
## part1. Concepts of OOP
- 객체지향프로그래밍(OOP)
	- data abstraction and encapsualtion
		- 클래스는 관련된 데이터와 그 데이터를 조작하기 위한 메서드들을 함께 묶어 놓음
		- 객체의 중요한 세부 정보는 숨김
		- 클래스 외부에서는 메서드들이 어떻게 동작하는지에 대한 세부 사항을 드러내지 않음
		- 이렇게 함으로써, 클래스는 ==데이터에 대한 추상화==를 제공하고, 개발자는 해당 클래스의 인터페이스를 통해 객체를 사용할 수 있음
	- Encapsulation
		- 클래스는 데이터와 메서드들을 하나로 묶어서 캡슐화하는데 사용됨
		- 데이터는 외부에서 직접 접근할 수 없도록 보통 private으로 선언되고 클래스 외부에서는 public으로 선언된 메서드를 통해 데이터에 접근하거나 조작할 수 있음
		- 이러한 메서드들은 클래스의 내부 구현 세부 사항을 캡슐화하고, ==데이터의 무결성과 일관성을 유지==함
- relationship between classes
	- association(use-a)
		- --------->
	- aggregation and composition(has-a)
		- ◇--------
	- inheritance(is-a)
		- ---------▷

## part2. Using Predefined Classes
- object reference variables
	- null reference
		- ==instance variable==은 초기화하지 않으면 ==기본값으로 초기화==됨
		- reference variable은 초기화하지 않으면 객체가 아직 할당되지 않은 상태로서 'null'로 초기화됨
	- uninitialized reference
		- ==local variable==은 초기화하지 않으면 ==compile-time error==가 발생함

## part3. User Defined Classes
- instance variable
	- default initial value가 있음
	- scope: ==static method를 제외==한 class 전반
- constructor
	- public이어야 함
	- 하나의 class는 여러 constructor를 가질 수 있음 (==overloading== -> ==polymorphism==의 한 종류)

## part4. More on Methods & Encapsulation
- java는 call by value
	- 객체 참조 변수를 매개변수로 가지는 method에서는 객체 참조 변수의 값만 복사해서 전달되기 때문에 객체 참조 변수를 직접 교환할 수 없음
- method overloading
	- same name, different sets of parameters(type, 개수, 순서)
	- constructor가 대표적
- encapsulation
	- implementation details는 숨기고 external behavior은 드러냄
	- 클래스의 method는 ==해당 클래스의 모든 객체의 private 변수==를 직접 조작할 수 있음

## part5. Final Variables & Static Variables and Methods
- final instance variable
	- final은 변수가 한 번 초기화되면 값을 변경할 수 없음을 보장
	- final이 참조하는 ==객체의 내부 상태==는 변경될 수 있음
- static method
	- = class method
	- static method는 ==non-static method( = object method)를 바로 부를 수 없음==

## part6. Object Construction
- instance field 초기화 3가지 방법
	- ==constructor==로 초기화
	- ==declaration==에서 초기화
		- method call로 초기화 가능
	- ==initialization block==에서 초기화
		- object가 만들어지면 자동으로 실행되는 code block
		- static initialization block은 class가 load될 때 실행됨
- the order of initialization
	- 첫 번째 constructor가 두 번 constructor를 호출하는 경우, 두 번째 constructor의 본문이 실행된 후 첫 번째 constructor의 본문이 마저 실행됨

## part7. More Class Examples
- helper function
	- 해당 클래스의 다른 method들을 도와주는 method, private으로 선언됨
- aggregation relationship
	- has-a relationship
	- 집과 방처럼 한 객체가 다른 객체를 포함하지만 종속적이지 않음
- composition relationship
	- part-of relationship
	- 차와 엔진처럼 한 객체가 다른 객체의 부분으로 구성되어 있고 부모 객체와 자식 객체는 생명 주기를 공유함

## part8. Packages
- class importation
	- * 사용: package내의 모든 class를 import, wildcard import
	- import하는 packages가 같은 class를 포함하는 경우
	  -> compile-time error
	  -> 한 package의 ==class를 specific하게 import==함으로써 해결 가능
- static imports
	- class의 static 멤버들을 클래스 이름을 지정하지 않고 직접 사용할 수 있게 해줌
- javac PackageTest.java
  java PackageTest
  -> PackageTest.java에 subdirectory의 class가 import 돼 있으면 그 class의 java 소스 코드를 compile하지 않아도 자동으로 찾아줌
- bin directory
	- java 소스 코드를 컴파일하면 '.class'파일로 변환되고 java프로젝트의 'bin'디렉터리에 저장되어 프로그램을 실행할 때 사용됨
