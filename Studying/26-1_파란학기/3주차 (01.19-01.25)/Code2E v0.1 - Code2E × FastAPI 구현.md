
## 실험 문제 정의 - 나눗셈 API

- 실패가 잘 나고, 테스트가 진실인 문제
	- `GET /divide?a=10&b=2` → `{"result": 5.0}`
	- `b=0`이면 400 에러 + 메시지
- 이 문제는:
	- 처음 생성 코드에서 `ZeroDivisionError`가 잘 터지고
	- 입력 검증/에러 처리로 수렴 루프가 생김
---

## 단계 1.  기본 틀 설계 (LLM 연결 전)

```
┌──────────────┐
│   main.py    │  ← 실행 엔트리
└──────┬───────┘
       ↓
┌────────────────────────┐
│      MasterAgent       │  ← 유일한 컨트롤러
│   (상태 + 반복 + 판단)    │
└──────┬─────────┬───────┘
       ↓         ↓
┌───────────┐   ┌───────────┐
│ CodeAgent │   │ TestAgent │   ← "행위자"
│ (코드 생성) │   │ (테스트 생성)│
└──────┬─── ┘   └───── ─┬───┘
       ↓               ↓
       └──────┬────────┘
              ↓
        ┌──────────────┐
        │ Execution    │  ← Truth (유일한 사실)
        │ (pytest 실행)│
        └──────┬───────┘
               ↓
        ┌──────────────┐
        │ DecisionLayer│  ← 규칙 기반 판단
        └──────┬───────┘
               ↓
        ┌──────────────┐
        │ FinalAgent   │  ← 보고서 생성
        └──────────────┘
```
