
## part1 
- types: primitive types, reference types
- C언어의 pointer, Java의 reference type
	- pointer는 임의의 메모리 주소 저장 가능
		- 잘못된 주소 저장 가능(segment fault)
	- reference type은 임의의 메모리 주소 저장 불가
	- new키워드로 생성한 객체를 선언하면 JVM이 대신 객체의 주소를 저장
	- reference type 사용 이유
		- garbage collector 때문에 객체의 주소 가변적
		- 잘못된 주소에 접근할 수 없게 하므로 안전함
- int를 float으로 못 바꿀 수도 있음
	- -> flaot이 유효자리숫자가 더 작음
- unicode: fixed-width 16-bit(2byte) characters
- string length에서 code points 와 code units 차이
	- ==code point==: ==모든 unicode character==가 하나씩 가짐
	- ==code unit==: 16-bit value, ==supplementary character는 2 code units==을 가짐
- ==char value 하나는 code unit 하나 표현== 가능
- supplementary character는 ==two-char value array 나 String 하나==로 표현 가능
- static:
	- 클래스에서 공유되는 변수나 메소드를 정의할 때 사용
	- 각 객체들에서 하나의 값이 공통적으로 값이 유지되어야 할 경우
	- 메모리의 static영역에 적재되고, 프로그램이 종료될 때 해제됨
- final: 한 번 정해지면 못 바꿈. 객체마다 다른 값은 가능
- static fianl: 상수 선언하고 싶을 때 사용
- String클래스 메소드
	- length() - codePointCount()
	- charAt() - offsetByCodePoints()
- string concatenation: 피연산자가 string이 아니면 string으로 바뀜

## part2
- Java는 내부적으로 UTF-16 인코딩 사용
- Exception Handling: throws IOException
- java 파일명은 main method를 가진 public class 명과 같아야 함
- 접근 제한자(access modifier):
  public, protected, pirvate, default
- classs의 접근 제한
	- public, default: 패키지 간 사용 가능, 불가능 차이
	- protected: 같은 패키지 + 다른 패키지에 속한 해당 클래스의 자식 클래스
- field와 method의 접근 제한
	- public: 모든 패키지
	- default: 같은 패키지 내부
	- private: 클래스 내부, 해당 클래스의 객체의 private 변수는 해당 클래스 내에서 마음껏 사용 가능
- field: class에 포함된 변수
	- class variable(static variable)
	- instance variable
- local variable
