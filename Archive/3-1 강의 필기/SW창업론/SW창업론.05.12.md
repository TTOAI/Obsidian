### 유저 플로우에 초점 맞추기
- 앱을 키면 바로 카메라 기능 켜짐
- 사진을 찍으면 바로 GPS 기반 위치 자동 기록
- 식당 이름, 해시태그, 음식 종류 짧게 기록 가능
- 즐겨찾기 추가 가능
- 앱 자체 캘린더 기능에 기록 추가 -> 나중에 찾아볼 수 있음
- 공유
	- 인스타 스토리로 바로 공유 가능 (NRC 처럼)
	- 구글 캘린더에 해당 기록을 등록 가능

### 우리동네 퓰리처상
- 인스타 공유를 한 사진 기록에 한해 우리동네 퓰리처상 후보 자동 등록
- 주 1회 우리동네 퓰리처상 시상

### 수익화
- 우리동네 퓰리처상의 상은 해당 음식점이나 제휴 음식점 기프티콘
- 트래픽 분석을 통해 언급이 가장 많이 된 음식점 중심으로 제휴 집중
- 이후 기존에 의논했던 리뷰 플랫폼으로 스케일 확장

### 기존 플랫폼의 API 사용
##### Instagram
- Graph API
	- 비즈니스·크리에이터 계정의 미디어 게시·댓글 관리·해시태그 검색 등을 지원
	- 해시태그 검색 기능을 이용하면 특정 키워드(#맛집, #음식점명 등)를 포함한 공개 포스트 조회 가능
	- 단, 새 해시태그는 7일간 최대 30개까지만 조회 가능
	- 텍스트(캡션) 분석을 통해 음식점 이름이나 메뉴 키워드를 추출하는 방법 고려 가능
- Basic Display API
	- `GET /me/media` 호출 결과에 각 미디어의 `permalink`(게시물 URL), `media_url`(이미지/동영상 URL), `thumbnail_url`(동영상 썸네일) 등이 포함
	- 캐러셀(다중 이미지) 게시물은 `children` 필드를 통해 하위 미디어 ID를 얻은 후 개별 호출

- Facebook 개발자 계정 → 앱 생성 → Instagram Login 사용 (OAuth)  
- Graph API: 비즈니스 인증, 앱 리뷰 필요 
- 호출 횟수 제한(예: 200회/시간 등)
- 해시태그 검색은 주 30개 제한

##### X
- X API v2
	- Free, Basic, Pro, Enterprise 티어로 나뉨
	- Free 티어
		- 글(트윗) 작성만 허용하며(월 1,500건) 읽기 전용 조회는 불가
	- Basic 티어(월 약 $100~200)
		- 약 1만1.5만건 수준의 트윗 읽기(검색 포함)와 5만건 게시
	- Pro 티어($5,000/월)
		- 백만 건 이상 조회, 실시간 필터 스트리밍(키워드 필터) 등 고급 기능 지원
	
	- X API의 검색(endpoint `/2/tweets/search/recent`)에 키워드나 해시태그를 쿼리
		-  해당 단어를 포함한 트윗을 얻을 수 있음
	- 위치 기반으로는 위도·경도와 반경을 지정해 트윗을 검색할 수 있음
		- 예) `point_radius` 쿼리
	- X API 호출 시 트윗 텍스트와 함께 이미지/동영상 미디어 URL을 얻을 수 있음

- X 개발자 계정 → 앱 생성 → OAuth/Bearer 토큰 필요  
- 무료 티어는 읽기 불가, Basic≥ 필요 (월 1–2만 건 조회, $100~200)
- 호출 한도(예: 검색 15분당 수백 건), 요금제별 월별 트윗 수 제한
