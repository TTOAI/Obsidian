
# 0. 주소 입력 직후 - 브라우저/로컬

- URL 정규화
    - `naver.com` 입력 → 브라우저가 자동으로 `https://naver.com` 시도
    - HSTS 프리로드 되어 있으면 HTTP 없이 바로 HTTPS만 허용

- 캐시/서비스워커 확인
    - 브라우저 HTTP 캐시, Service Worker가 있으면 응답을 로컬에서 제공할 수도 있음

- 네트워크 준비
    - IP를 아직 모르면 DNS 필요
    - 외부로 나가기 전에 게이트웨이 MAC 주소를 ARP(IPv4)/ND(IPv6)로 확인
		- (참고) 게이트웨이:
			- 집/회사 네트워크에서 인터넷으로 나가는 출입문 역할을 하는 장치
			- 보통 공유기/라우터
		- (참고) MAC 주소:
			- 네트워크 카드(NIC)에 할당된 물리적 식별자
		- (참고) ARP(Address Resolution Protocol):
			- 게이트웨이의 MAC 주소를 알기 위해 브로드캐스트를 수행하는 프로토콜
		- (참고) DHCP(Dynamic Host Configuration Protocol):
			- PC/스마트폰이 네트워크에 붙으면 DHCP 서버(보통 공유기)가 다음 정보를 제공
				- 내 IP 주소
				- 서브넷 마스크
				- 기본 게이트웨이 IP 주소
				- DNS 서버 주소
			- 처음부터 게이트웨이 MAC 주소를 주지 않는 이유:
				- 네트워크 계층 분리 원칙에 따라 3계층 프로토콜인 DHCP에서 2계층의 세부 정보인 MAC 주소를 직접 제공하지 않음
				- 게이트웨이 IP 뒤에 여러 장치가 존재하거나 장치가 바뀔 수 있음
    - 가정/회사 라우터를 통해 NAT 거쳐 공인망으로 나갈 준비
