### Selenium의 한계

- `element click intercepted` 오류 발생
```
Message: element click intercepted: Element <a href="#none" class="link_refresh" data-tiara-layer="refresh" data-tiara-ordnum="3">...</a> is not clickable at point (432, 19). Other element would receive the click: <span class="txt_lnb null">...</span>
```
- 클릭하려는 새로고침 버튼(`link_refresh`) 위에 다른 요소(`span.txt_lnb`)가 시각적으로 겹쳐 있어서 클릭 이벤트가 가로막힌 상황

- Selenium의 클릭 동작 방식의 한계
	- OS 수준의 마우스 이벤트 시뮬레이션에서 다른 요소가 덮으면 `ElementClickInterceptedException` 발생

- Playwright의 클릭 동장 방식
	- JS 레벨에서 클릭 이벤트를 직접 dispatch → 덮여 있어도 내부적으로 click 실행 가능

- 이 외의 Selenium의 한계
	- 렌더링 인식
		- DOM 업데이트와 뷰포트 스크롤 상태를 직접 감시하지 못함
	- 자동 스크롤 보정 없음

### Git Bash에서 Playwright의 Chromium 설치가 안 되는 이유

- `python -m playwright install chromium`을 실행
- Node.js 런타임 기반의 내부 CLI(playwright-cli.exe)를 실행
- 이 CLI가 C로 작성된 압축 해제 로직(libarchive, zlib 등)을 통해 Chromium 바이너리를 다운로드 및 압축 해제
- Windows의 로컬 사용자 경로 (AppData\Local\ms-playwright)에 저장
- git bash의 모든 명령은 MSYS2 가상 DLL을 통해 중간 변환 후 실행
- Playwright가 Chromium을 설치하려고 Win32 경로(`C:\Users\USER\AppData\Local...`)를 직접 열 때, MSYS2가 경로를 변환해버리거나 DLL 핸들을 잘못 넘겨서 세그폴트(segmentation fault)가 발생
- 결과적으로 MSYS2 하위호환성 문제 때문
- Git Bash 대신 PowerShell 또는 CMD에서 설치해야 함

### Playwright에서 Scrapy-Playwright로 migration

- Scrapy는 비동기(Async I/O) + 병렬 구조

- Scrapy-Playwright:
	- Playwright 엔진을 Scrapy 내부에서 호출
	- 비동기(asyncio) 기반 고속 병렬 처리

- Scrapy-Playwright 동작 과정
```
Spider (yield item)
   ↓
Item Pipeline (데이터 후처리: 정제, 중복검사, 저장 등)
   ↓
Feed Exporter (파일/DB/S3로 자동 저장)
```

- 하지만 migration 중 막힘 -> Playwright 유지
