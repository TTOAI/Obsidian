
### Python 프로젝트에서 의존성, 패키징 관리와 관련된 파일

- requirements.txt
- environment.yml
- setup.py

### requirements.txt

- 가장 기본적인 파일 형식
- 프로젝트를 실행하는 데 필요한 Python 패키지들의 목록을 정의
- 주로 **`pip`** 패키지 관리자를 위해 사용
- 패키지 이름과 버전 제약 조건을 명시하는 데 중점을 둠
- 가상 환경을 설정하는 기능은 없으며, 단순히 필요한 패키지를 나열

### environment.yml

- `Conda` 패키지 관리자(`Anaconda`/`Miniconda`)를 사용하여 재현 가능한 환경 전체를 정의
- Python 버전, 패키지 목록, 채널(channel), C/C++ 라이브러리까지 포함하여 환경을 완벽하게 정의 가능
	- 채널(channel)
		- 기본적으로 Conda 패키지 파일(`tar.bz2` 또는 `conda.yaml` 파일)이 저장되어 있는 온라인 저장소(Repository) 또는 디렉토리
		- `defaults` (기본 채널)
			- Anaconda가 기본으로 제공하는 채널
			- 대부분의 표준적이고 안정적인 과학 컴퓨팅 패키지(Numpy, Pandas 등) 포함
		- `conda-forge` (커뮤니티 채널)
			- 커뮤니티 주도로 관리되는 매우 인기 있는 채널
			- 다양한 최신 패키지를 제공
- `conda` 환경을 생성하고 활성화하는 데 사용

### setup.py

- Python 패키지를 빌드하고 배포하기 위한 파일
- `setuptools` 라이브러리를 사용하여 프로젝트를 설치 가능한 패키지(예: `.whl`, `.tar.gz` 파일)로 만듦
- 패키지의 메타데이터(이름, 버전, 작성자 등)와 실행에 필요한 의존성(`install_requires`)을 정의함
- 프로젝트를 PyPI(Python Package Index)에 업로드하거나 다른 프로젝트에서 `pip install`로 직접 설치할 수 있도록 만들어 줌
- 배포용 패키지의 정의가 주 목적
