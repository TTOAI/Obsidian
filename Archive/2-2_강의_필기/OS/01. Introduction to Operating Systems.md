
# OS

- user/application과 computer hardware 사이의 중재자 역할
- hardware resources를 제어 및 조정
- Protects the system (from the system’s standpoint) and serves applications (from the application’s standpoint)
- interrupt을 감지했을 때 interrupt를 처리하고 시스템을 last interrupted state로 되돌림

# Computer-System Architecture

- Asymmetric Multiprocess
	- 각각의 CPU가 특정한 task를 처리함 (Boss-worker relationship)
vs
- Symmetric Multiprocessor (SMP)
	- 각각의 CPU가 모든 task를 처리함
	- 각각의 CPU가 single core chip을 구성함
vs
- Single-chip multiprocessor (a.k.a. multicore processor)
	- 공유된 자원(cache, memory 등)을 사용하고 core들 간의 거리가 짧음

- Uniform Memory Access architecture (UMA)
	- 모든 CPU가 단일 메모리에 접근
	- 동일한 메모리 접근 속도
vs
- Non-Uniform Memory Access architecture (NUMA)
	- 여러 CPU가 개별 메모리를 할당 받음
	- 다른 CPU의 메모리에도 접근 가능
	- CPU가 늘어나도, 각 CPU가 자신의 메모리를 효율적으로 사용할 수 있어 확장성이 뛰어남

# I/O structure

- device controller
	- 각각의 device type(키보드, 마우스, 하드 디스크 등)을 직접 제어하는 하드웨어 구성 요소
	- 각 장치마다 고유한 특징을 가지고 있으며, 이에 맞는 명령어 집합(레지스터)을 가지고 있음
	- 각각의 local buffer를 가짐
	- device와 buffer 사이에서 data를 전달함
	- I/O transfer이 종료되었을 때 device driver에 알림
	- interrupt를 생성하여 CPU에게 event를 알림
		- I/O device로부터 request를 받았을 때
		- I/O operation이 종료되었을 때
	- CPU는 interrupt-request lines을 통해 interrupt를 감지함
- device handler
	- CPU가 interrupt를 감지했을 때 실행하는 함수
	- CPU는 전달 받은 interrupt ID로 table에서 그에 맞는 interrupt handler를 찾음
		- interrupt descriptor table
		- interrupt vector Table
- device driver
	- OS와 device controller 사이에서 중개 역할을 하는 소프트웨어
	- 다양한 device controller의 특성을 추상화하여 OS에게 간단하고 일관된 인터페이스를 제공
	- I/O operation을 시작하기 위해 디바이스 컨트롤러 내부의 레지스터를 설정
		- 디바이스 컨트롤러는 설정된 레지스터 값에 따라 작업을 수행
