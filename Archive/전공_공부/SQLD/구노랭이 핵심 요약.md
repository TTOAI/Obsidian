- SQL server에서는 AUTO COMMIT을 FALSE로 설정할 경우 DDL을 실행하더라도 자동으로 commit되지 않음

- [ORACLE] ROLLBACK
	- SAVEPOINT SVPT1;
	- ...
	- ROLLBACK TO SVPT1;

-  [SQL server] ROLLBACK
	- SAVE TRANSACTION SVPT1;
	- ...
	- ROLLBACK TRANSACTION SVPT1;

- NULL의 연산
	- NULL과 (+, -, \*, /) -> NULL
	- NULL과 (=, <, <=, >, >=) -> NULL

- ''의 인식
	- ORACLE -> NULL로 인식
	- SQL server -> 공백문자 데이터로 인식

- TO_DATE()
	- 특정 시점으로 바뀜 -> 기간 X

- [ORACLE] 날짜형 데이터
	- + 1 -> 하루가 더해짐
	- 1/24(60/10) = 10분

- simple case와 searched case
	- simple case
		- CASE color WHEN 'red' THEN 'STOP'
		- ELSE "UNKNOWN"
		- END
	- searched case
		- CASE WHEN score >= 90 THEN 'A'
		- ELSE 'F'
		- END

- [ORACLE] NVL(표현식, 다른 값)
	- 표현식 값이 NULL일 경우 다른 값을 반환

- - [SQL server] ISNULL(표현식, 다른 값)
	- 표현식 값이 NULL일 경우 다른 값을 반환

- 생략되었을 경우 보통 NULL로 생각

- NULLIF(a, b)
	- a = b일 경우 NULL, 아닐 경우 a

- COALESCE(표현식, ...)
	- NULL이 아닌 첫 번째 값을 반환

- GROUP BY에서 NULL 그룹이 있으며 count에 포함

- AVG(COUNT(\*)) -> X

- ORDER BY절의 기준이 GROUP BY절의 기준도 아니고 SELECT절에 존재하지도 않음
  -> 오류

- [ORACLE] NULL의 크기
	- 가장 큰 값으로 간주

- [SQL server] NULL의 크기
	- 가장 작은 값으로 간주

- FROM - WHERE - GROUP BY - HAVING - SELECT - ORDER BY

- USING과 ON
	- JOIN 뒤에 사용하여 JOIN 조건을 명시함
	- USING
		- 테이블 간에 동일한 이름을 가진 열을 이용
	- ON
		- 두 테이블 간의 관계를 정의

- UNION과 UNION ALL
	- UNION은 내부적으로 정렬 과정과 DISTINCT과정을 거침
		- 중복을 제거함
		- 성능이 더 느림

- FROM 절 서브쿼리
	- = 인라인 뷰, 동적 뷰
